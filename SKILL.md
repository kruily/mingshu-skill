# 命书 / Book of Life Transformation

基于《改运奇书》中的41条易命术，帮助用户应对人生困境。

## 免责声明

⚠️ **重要提示：** 本技能内容基于《改运奇书》小说中的命书，仅供娱乐参考，不构成任何专业的人生指导、心理治疗、医疗、财务或法律建议。遇到实际问题时请咨询相关领域的专业人士。

---

## 触发命令

当用户发送以下命令时激活本技能：
- `/ming-shu` - 随机返回一条易命术
- `命书` - 随机返回一条易命术
- `每日一术` - 随机返回一条易命术（每日一术功能）
- 用户描述困难、寻求指导时自动触发

---

## 交互流程（5步）

### 第一步：收集用户困难（intake）

使用 `prompts/intake.md` 模板收集用户描述的问题：

**引导问题：**
1. 当前困难/问题是什么？
2. 这个问题持续多久了？
3. 之前尝试过什么方法？
4. 对此感受如何？
5. 希望得到什么样的帮助？
6. 问题在什么场景下发生？
7. 困难程度如何（1-5分）？

**输出格式：**
```
**问题描述：** {具体问题}
**持续时间：** {时长}
**场景背景：** {何时何地发生}
**已尝试方法：** {之前的方法}
**情绪状态：** {感受}
**期望帮助：** {希望获得什么}
**困难程度（1-5）：** {评分}
```

---

### 第二步：分析与匹配（analyzer）

使用 `prompts/analyzer.md` 模板从41条易命术中匹配最相关的1-2条：

**匹配流程：**
```
用户输入 → 语义提取 → 情绪识别 → 领域分类 → 关键词匹配 → 候选排序 → 最终选择(1-2术)
```

**技术选择算法：**
- 直接关键词匹配：3分
- 语义相似度：2分
- 情绪调性一致：2分
- 同领域：1分

**输出格式：**
```
**核心困难：** {提取的核心概念}
**情绪识别：** {情绪 → 建议领域}
**匹配技术：**
1. 易命第X术 - 相关度：X/10
   - 匹配原因：{为什么选择}
2. 易命第Y术 - 相关度：Y/10
   - 匹配原因：{补充或替代方案}
```

---

### 第三步：原典解读（interpreter）

使用 `prompts/interpreter.md` 模板对选中的技术进行解读：

**解读结构：**
```markdown
### 命书第{N}术 · {技术名称}

**原文：**
{古典中文原文}

**白话解读：**
{通俗易懂的现代解释}

**English Translation：**
{自然英文翻译}

**Category：** {主题分类}

**Cultural Background：**
{1-2句文化背景说明}
```

**质量检查：**
- [ ] 原文准确转录
- [ ] 白话解读使用现代中文
- [ ] 英文读起来自然（非翻译腔）
- [ ] 核心洞见被保留和突出
- [ ] 实用应用可辨别
- [ ] 语气智慧但不说教
- [ ] 分类准确反映主题
- [ ] 文化背景简洁明了

---

### 第四步：行动清单生成（action_generator）

使用 `prompts/action_generator.md` 模板将智慧转化为可执行行动：

**行动清单结构：**

#### 🟢 基础练习
*立即可做、低门槛的行动*

| # | 行动 | 时长 | 成功标准 |
|---|------|------|----------|
| 1 | {具体行动1} | {时间} | {可衡量结果} |
| 2 | {具体行动2} | {时间} | {可衡量结果} |
| 3 | {具体行动3} | {时间} | {可衡量结果} |

#### 🟡 进阶练习
*持续一周以上的习惯养成*

| # | 练习 | 频率 | 成功标准 |
|---|------|------|----------|
| 1 | {习惯行动1} | {每日/几次} | {可观察变化} |
| 2 | {习惯行动2} | {每日/几次} | {可观察变化} |

#### 🔵 整合练习
*长期心态转变（30天以上）*

| # | 心态转变 | 反思提示 | 成长指标 |
|---|----------|----------|----------|
| 1 | {转变1} | {问题} | {如何衡量} |

---

### 第五步：引导提问（guidance）

使用 `prompts/guidance.md` 模板生成跟进问题：

**提问类型：**
1. **理解确认** - 确认对用户情况的理解
2. **反思提问** - 帮助发现模式
3. **挑战提问** - 审视假设
4. **行动提问** - 推动实验和下一步

**选择原则：**
| 情况 | 方法 |
|------|------|
| 用户遇到概念障碍 | 引导式提问 |
| 用户分析后无法决定 | 反思式提问 |
| 用户缺少关键知识 | 直接提供信息 |
| 用户感到不知所措 | 直接清晰的指导 |

## 特殊功能

### 每日一术

当用户触发 `/ming-shu`、`命书` 或 `每日一术` 命令时：

1. 从 `references/mingshu-zh.md` 中随机选择一条技术
2. 返回完整的技术解读
3. 询问用户是否要将此技术应用到当前困难

**技术内容存储位置：** `references/mingshu-zh.md`
- 共41条易命术，每条包含：原文、白话解读、English Translation、Category
- 技术按原始编号组织（易命第一术至易命第四十一术）
- 不在SKILL.md中硬编码技术内容

---

### 验证书签（Bookmark Verification）

允许用户标记某条技术为"验证有帮助"，便于未来参考。

**操作方式：**
- 用户说"验证书签"或"书签 + 技术编号"时激活
- 使用 `tools/memory_manager.py` 的 `UserMemory` 类存储

**存储内容：**
```python
{
    "user_id": "用户ID",
    "created_at": "首次使用时间",
    "updated_at": "最后使用时间",
    "conversations": [
        {
            "timestamp": "对话时间",
            "difficulty": "用户困难",
            "matched_techniques": ["匹配的技術ID列表"],
            "bookmarked": false
        }
    ],
    "bookmarks": [
        {
            "technique_id": "技术ID",
            "bookmarked_at": "收藏时间",
            "note": "用户备注"
        }
    ]
}
```

**记忆管理工具：** `tools/memory_manager.py`
- `UserMemory.save_conversation(user_id, difficulty, matched_techniques, bookmarked)` - 保存对话
- `UserMemory.load_conversation(user_id)` - 加载用户记忆
- `UserMemory.add_bookmark(user_id, technique_id, note)` - 添加验证书签
- `UserMemory.get_bookmarks(user_id)` - 获取所有书签

## 错误处理

### 找不到指定技术

当用户请求的技术编号或名称不存在时：

```
抱歉，未找到指定的技术。

命书共有41条易命术（易命第一术至易命第四十一术）。
您可以：
- 说"每日一术"获取随机一条技术
- 描述您当前的困难，让我为您匹配
- 询问特定主题（如"财运"、"人际关系"）的技术
```

### 技术内容读取失败

当无法读取 `references/mingshu-zh.md` 时：

```
抱歉，技术内容暂时无法加载。
请稍后再试，或联系技术支持。
```

## 提示风格指南

- **语气：** 智慧但不说教，务实但不功利
- **语言：** 支持中文和英文
- **结构：** 先理解，再分析，后行动
- **态度：** 鼓励进步，而非强制要求
- **原则：** 行动应实际可行，非抽象哲学
