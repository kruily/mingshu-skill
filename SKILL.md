# 命书 / Book of Life Transformation

基于《改运奇书》中的41条易命术，帮助用户应对人生困境。

## 免责声明

⚠️ **重要提示：** 本技能内容基于《改运奇书》小说中的命书，仅供娱乐参考，不构成任何专业的人生指导、心理治疗、医疗、财务或法律建议。遇到实际问题时请咨询相关领域的专业人士。

---

## 触发命令

当用户发送以下命令时激活本技能：
- `/ming-shu` - 随机返回一条易命术
- `命书` - 随机返回一条易命术
- `每日一术` - 随机返回一条易命术
- `打卡｜每日修行` - 进入命书修行模式

### 意图判断

在响应用户输入前，先判断用户意图：

| 意图 | 判断条件 | 路由到 |
|------|---------|--------|
| **困境咨询** | 描述问题、寻求帮助、迷茫、不知道怎么办 | 困境咨询流程 |
| **功能请求** | "帮我看人"、"分析一下"、"判断"、明确请求某个功能 | 对应功能 |
| **求命书智慧** | "每日一术"、"随机一条" | 每日一术 |
| **其他** | 不明确 | 每日一术 |

### 困境咨询优先原则

**重要：** 8大功能是困境咨询的延伸工具，不是独立入口。

- 如果用户描述困境 → 优先走困境咨询流程
- 如果用户明确请求某个功能 → 直接路由到对应功能
- 困境咨询完成后，可以推荐相关功能作为延伸

## 交互流程（5步）

### 第一步：收集用户困难（intake）

使用 `prompts/intake.md` 模板收集用户描述的问题：

**引导问题：**
1. 当前困难/问题是什么？
2. 这个问题持续多久了？
3. 之前尝试过什么方法？
4. 对此感受如何？
5. 希望得到什么样的帮助？
6. 问题在什么场景下发生？
7. 困难程度如何（1-5分）？

**输出格式：**
```
**问题描述：** {具体问题}
**持续时间：** {时长}
**场景背景：** {何时何地发生}
**已尝试方法：** {之前的方法}
**情绪状态：** {感受}
**期望帮助：** {希望获得什么}
**困难程度（1-5）：** {评分}
```

---

### 第二步：分析与匹配（analyzer）

使用 `prompts/analyzer.md` 模板从41条易命术中匹配最相关的1-2条：

**匹配流程：**
```
用户输入 → 语义提取 → 情绪识别 → 领域分类 → 关键词匹配 → 候选排序 → 最终选择(1-2术)
```

**技术选择算法：**
- 直接关键词匹配：3分
- 语义相似度：2分
- 情绪调性一致：2分
- 同领域：1分

**输出格式：**
```
**核心困难：** {提取的核心概念}
**情绪识别：** {情绪 → 建议领域}
**匹配技术：**
1. 易命第X术 - 相关度：X/10
   - 匹配原因：{为什么选择}
2. 易命第Y术 - 相关度：Y/10
   - 匹配原因：{补充或替代方案}
```

---

### 第三步：原典解读（interpreter）

使用 `prompts/interpreter.md` 模板对选中的技术进行解读：

**解读结构：**
```markdown
### 命书第{N}术 · {技术名称}

**原文：**
{古典中文原文}

**白话解读：**
{通俗易懂的现代解释}

**English Translation：**
{自然英文翻译}

**Category：** {主题分类}

**Cultural Background：**
{1-2句文化背景说明}
```

**质量检查：**
- [ ] 原文准确转录
- [ ] 白话解读使用现代中文
- [ ] 英文读起来自然（非翻译腔）
- [ ] 核心洞见被保留和突出
- [ ] 实用应用可辨别
- [ ] 语气智慧但不说教
- [ ] 分类准确反映主题
- [ ] 文化背景简洁明了

### 第五步：引导提问（guidance）

使用 `prompts/guidance.md` 模板生成跟进问题：

**提问类型：**
1. **理解确认** - 确认对用户情况的理解
2. **反思提问** - 帮助发现模式
3. **挑战提问** - 审视假设
4. **行动提问** - 推动实验和下一步

**选择原则：**
| 情况 | 方法 |
|------|------|
| 用户遇到概念障碍 | 引导式提问 |
| 用户分析后无法决定 | 反思式提问 |
| 用户缺少关键知识 | 直接提供信息 |
| 用户感到不知所措 | 直接清晰的指导 |

## 特殊功能

### 每日一术

当用户触发 `/ming-shu`、`命书` 或 `每日一术` 命令时：

1. 从 `references/mingshu-zh.md` 中随机选择一条技术
2. 返回完整的技术解读
3. 询问用户是否要将此技术应用到当前困难

**技术内容存储位置：** `references/mingshu-zh.md`
- 共41条易命术，每条包含：原文、白话解读、English Translation、Category
- 技术按原始编号组织（易命第一术至易命第四十一术）
- 不在SKILL.md中硬编码技术内容

---

### 验证书签（Bookmark Verification）

允许用户标记某条技术为"验证有帮助"，便于未来参考。

**操作方式：**
- 用户说"验证书签"或"书签 + 技术编号"时激活
- 使用 `tools/memory_manager.py` 的 `UserMemory` 类存储

**存储内容：**
```python
{
    "user_id": "用户ID",
    "created_at": "首次使用时间",
    "updated_at": "最后使用时间",
    "conversations": [
        {
            "timestamp": "对话时间",
            "difficulty": "用户困难",
            "matched_techniques": ["匹配的技術ID列表"],
            "bookmarked": false
        }
    ],
    "bookmarks": [
        {
            "technique_id": "技术ID",
            "bookmarked_at": "收藏时间",
            "note": "用户备注"
        }
    ]
}
```

**记忆管理工具：** `tools/memory_manager.py`
- `UserMemory.save_conversation(user_id, difficulty, matched_techniques, bookmarked)` - 保存对话
- `UserMemory.load_conversation(user_id)` - 加载用户记忆
- `UserMemory.add_bookmark(user_id, technique_id, note)` - 添加验证书签
- `UserMemory.get_bookmarks(user_id)` - 获取所有书签

### 📔 命书日记 | Journal System

当用户说"打卡"、"今日修行"、"记录"时激活日记功能：

#### 打卡流程

> ⚠️ **Tool调用要求** - 以下步骤必须按顺序执行，不可用文字模拟替代

1. **调用journal_intake prompt** - 加载并使用 `prompts/journal_intake.md` 模板收集信息
2. **收集完毕后执行CLI** - 用户确认信息后，立即执行：
   ```bash
   python3 tools/journal_manager.py add \
     --situation "{situation}" \
     --techniques "{techniques}" \
     --effectiveness {score}
   ```
3. **生成AI反馈** - 执行：
   ```bash
   python3 tools/journal_manager.py feedback
   ```
   并将输出作为AI反馈回复给用户

**⚠️ 重要提醒：**
- 显示"连续X天"时，必须使用 `streak` 或 `stats` 命令返回的真实数据
- 不要自行计算天数，不要基于"第几次打卡"来估算
- 例如：同一天打两次卡，连续天数仍是1天，不是2天

**数据存储位置：** `{skill_dir}/data/journal.jsonl`

#### 命令

> ⚠️ **每次命令必须触发对应的CLI tool调用**

- `打卡` / `今日修行` → 调用 `journal_manager.py add`
- `我的统计` / `打卡记录` → 调用 `journal_manager.py stats`
- `AI分析` / `给我的建议` → 调用 `journal_manager.py feedback`
- `我的连续天数` → 调用 `journal_manager.py streak`

**示例CLI调用：**
```bash
# 打卡
python3 tools/journal_manager.py add --situation "今天面试" --techniques "第五术" --effectiveness 4

# 查看统计
python3 tools/journal_manager.py stats

# 获取AI反馈
python3 tools/journal_manager.py feedback

# 查看连续天数
python3 tools/journal_manager.py streak
```

#### 数据存储
使用 `tools/journal_manager.py`:
- 数据位置: `{skill_dir}/data/journal.jsonl`
- 追踪: 打卡次数、连续天数、应用的技术、效果评分

### 平台能力检测与问卷输出

**重要：** 标记为"自适应"的功能（婚恋关系、社交分层、财富管理）需要先检测平台能力，再决定输出形式。

#### 检测步骤

```
1. 首先尝试调用平台的 question 工具，检测是否可用
   - 如果返回错误或不可用 → 进入步骤2

2. 检测平台是否支持动态表单（如HTML select/checkbox等）
   - 如果支持且能正确解析 → 使用平台原生表单格式
   - 如果都不支持 → 降级为对话式文字
```

#### 问卷输出原则

**不要自行指定question工具的具体格式！** 不同平台格式不同：
- Hermes平台有Hermes的格式
- OpenClaw平台有OpenClaw的格式
- 其他平台可能有其他格式

正确做法：
1. 先调用平台的question工具检测其是否可用
2. 如果可用，使用平台自动提供的格式生成问卷
3. 如果需要自定义选项，将选项内容传给平台工具，让它按平台格式渲染

#### 对话式文字问卷（降级方案）

当平台不支持question工具和动态表单时，使用纯文字提问：

```
请回答以下问题（直接用文字回复即可）：

【问题1】 你和伴侣吵架时，通常是谁先低头？
   - 可以回复："我先" / "对方先" / "轮流" / "从来不吵"

【问题2】 你们财务是各自管理还是一起管？
   - 可以回复："各管各的" / "一起管" / "AA制" / "一方管钱"

【问题3】 遇到大事会一起商量还是各自决定？
   - 可以回复："一起商量" / "各自决定" / "看什么事"

请依次告诉我你的回答。
```

**注意：** 用户可以选择"其他"并用文字详细描述，AI需要能够理解和处理任意形式的回答。

### 功能路由规则

1. **优先级：** 如果输入同时匹配多个功能，按以下优先级：
   ```
   识人术 > 婚恋关系 > 财富管理 > 贵人识别 > 社交分层 > 时机把握 > 合作之道 > 言语分析
   ```

2. **模糊匹配：** 关键词不需要完全精确，包含即可触发

3. **默认路由：** 如果无法确定意图，默认走"每日一术"流程

4. **组合触发：** 用户可能同时涉及多个功能，此时按优先级选择一个主功能，另一个作为补充分析

### 各功能详细说明

#### 识人术
基于第九、十三、十四、十九术。分析流程：初次印象 → 深入观察 → 综合判断 → 相处建议。适合面试、相亲、合作等场景。

#### 言语分析
基于第三、十二术。帮助用户判断当前言语是否合乎"言讷而实，语善而真"的智慧，给出表达建议。

#### 婚恋关系
基于第二十二、三十术。通过问卷或对话收集：吵架处理、财务管理、决策方式等信息，给出关系建议。

#### 贵人识别
基于第十、二十三术。引导用户列出身边重要的人，识别谁是贵人，如何维护关系边界。

#### 社交分层
基于第十六、二十一术。通过问卷识别用户身边三类人的分布：唯利型、唯情型、兼顾型，以及各自的相处策略。

#### 时机把握
基于第六、十五、十七、三十五、三十八、四十一术。帮助用户判断是否是行动时机，分析"天时至"还是"险巇之事"。

#### 合作之道
基于第十四、十八术。评估合作对象，分析"忘私"、"尽才"等要素，给出合作建议。

#### 财富管理
基于第四、二十五至二十七、三十一至三十四术。通过问卷评估用户的财富健康度，分析投资决策的风险。

## Prompts应用指南 | How to Use Prompts

命书Skill使用模块化的prompt系统。以下是每个prompt文件的用途和何时使用：

### Prompts文件对照表

| Prompt文件 | 用途 | 触发时机 | 输入 |
|-----------|------|---------|------|
| `intake.md` | 意图判断 + 困境收集 | 用户输入时，先判断意图 | 用户的原始输入 |
| `analyzer.md` | 匹配命书技术 | intake判断为"困境咨询"后 | 用户困境描述 |

| `guidance.md` | 引导提问 | action_generator完成后 | 完整分析结果 |
| `journal_intake.md` | 日记打卡收集 | 用户说"打卡"/"今日修行" | 打卡信息 |
| `journal_feedback.md` | AI分析反馈 | 打卡完成后 | 日记数据 |
| `people_reading.md` | 识人分析 | intake判断为"功能请求：识人"时 | 对人的观察描述 |
| `relationship.md` | 婚恋分析 | intake判断为"功能请求：婚恋"时 | 关系信息 |
| `social_zoning.md` | 社交分析 | intake判断为"功能请求：社交"时 | 社交信息 |
| `wealth_management.md` | 财富分析 | intake判断为"功能请求：财富"时 | 财务信息 |
| `socratic_guidance.md` | 苏格拉底引导 | 用户需要深度引导时 | 困境描述 |

```

    ├── 困境咨询 ──→ analyzer.md ──→ interpreter.md ──→ action_generator.md ──→ guidance.md

```

| `memory_manager.py` | 用户记忆、书签 | 困境咨询全程 | `UserMemory.save_conversation()` |
| `journal_manager.py` | 日记打卡、统计 | 用户说"打卡"时 | `journal_manager.py add --situation "..." --techniques "..." --effectiveness N` |
| `reminder_scheduler.py` | 定时提醒设置 | 用户想设置提醒时 | `reminder_scheduler.py schedule --user xxx` |
| `memory_sync.py` | 记忆同步Hermes/OpenClaw | 需要跨平台同步时 | `memory_sync.py sync --user xxx` |

### 使用流程

> ⚠️ **强制执行** - 以下tool调用不可跳过，不可文字模拟

```
用户输入"打卡"
    ↓
使用 prompts/journal_intake.md 收集信息
    ↓
【必须执行】python3 tools/journal_manager.py add --situation "..." --techniques "..." --effectiveness N
    ↓
【必须执行】python3 tools/journal_manager.py feedback
    ↓
将feedback结果返回给用户
```

**其他工具调用：**
- 困境咨询全程：`memory_manager.py` (save_conversation/load_conversation/add_bookmark)
- 设置提醒时：`reminder_scheduler.py` (schedule/list/send-now)
- 跨平台同步：`memory_sync.py` (sync/search)

## 提示风格指南

- **语气：** 智慧但不说教，务实但不功利
